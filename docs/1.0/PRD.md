# 头像预览微信小程序 · 需求规划文档

> 版本：v1.0 · 日期：2026-03-08

---

## 一、产品概述

### 产品定位
一款帮助用户在**更换微信头像前**预览实际效果的工具型小程序。用户上传候选头像图片后，可在高度还原的微信界面模拟中查看新头像的真实呈现效果，降低换头像的决策成本。

### 目标用户
有更换微信头像需求、但不确定新头像在各场景下效果是否好看的微信用户。

### 核心价值
- 所见即所得，换头像前先"试穿"
- 无需登录，打开即用，零门槛
- 覆盖多个真实微信使用场景，预览全面

## 二、功能需求

### 2.1 图片上传与裁剪

| 功能点 | 说明 |
|--------|------|
| 本地图片上传 | 支持从相册选择图片 |
| 裁剪功能 | 支持自由裁剪区域选择 |
| 缩放功能 | 支持双指缩放调整图片大小 |
| 圆形预览 | 裁剪框默认显示圆形遮罩（模拟微信头像展示形态） |
| 重新上传 | 支持随时更换图片重新裁剪 |

### 2.2 预览场景

小程序需还原以下 **4 个微信界面场景**：

#### 场景 1：单聊对话界面（自己视角）
- 模拟自己发送消息时的界面
- 自己的头像显示在右侧
- 支持自定义：昵称、消息内容

#### 场景 2：单聊对话界面（对方视角）
- 模拟对方看到自己消息时的界面
- 自己的头像显示在左侧
- 支持自定义：昵称、消息内容

#### 场景 3：朋友圈动态界面
- 模拟发布朋友圈后的效果
- 展示动态封面头像、昵称、发布内容、点赞/评论区
- 支持自定义：昵称、朋友圈文字内容

#### 场景 4：微信个人主页
- 模拟个人资料页的头像展示效果
- 支持自定义：昵称、微信号、个性签名

### 2.3 自定义信息

用户可在预览前填写以下个性化信息（全部为选填，不填则显示默认占位内容）：

| 字段 | 适用场景 | 默认值 |
|------|----------|--------|
| 昵称 | 所有场景 | "微信用户" |
| 消息内容 | 单聊 | "你好，这是一条消息～" |
| 朋友圈内容 | 朋友圈 | "分享一张美好的照片 🌟" |
| 个性签名 | 个人主页 | "这个人很神秘" |

### 2.4 用户系统

- **无需登录**，完全匿名使用
- 不存储用户上传的图片（纯前端处理）
- 如后续引入会员/付费功能，届时再接入微信授权登录

## 三、页面结构

```
小程序
├── 首页（上传入口）
│   ├── 上传头像按钮
│   └── 功能介绍引导
│
├── 裁剪页
│   ├── 图片裁剪/缩放操作区
│   ├── 圆形预览框
│   └── 确认/重选按钮
│
├── 信息填写页（选填）

│   ├── 各场景自定义内容输入

│
├── 预览主页
│   ├── 场景切换 Tab（4个场景）
│   └── 场景预览区（高仿微信 UI）

    └── 产品介绍 / 反馈入口
```

## 四、非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 图片裁剪与预览渲染流畅，无明显卡顿 |
| 兼容性 | 兼容 iOS / Android 主流机型 |
| 隐私安全 | 图片仅在本地处理，不上传服务器 |
| UI 风格 | 高度还原微信原生界面视觉风格 |
| 包体积 | 控制在微信小程序限制（2MB 主包）内 |

---

## 五、技术要点提示

- 图片裁剪可使用 `we-cropper` 等成熟的小程序裁剪组件
- 预览界面建议使用 Canvas 或纯 WXML/WXSS 还原微信 UI
- 图片处理全部在前端完成，无需后端服务（MVP 阶段）
- 注意微信小程序对 `wx.chooseMedia` 的图片格式和大小限制

---

## 六、MVP 开发优先级

| 优先级 | 功能模块 |
|--------|----------|
| P0（必须） | 图片上传、裁剪缩放、单聊预览（自己视角） |
| P1（重要） | 朋友圈预览、对方视角单聊 |
| P2（增强） | 个人主页预览、自定义信息填写 |
| P3（后续） | 保存预览截图、登录与会员体系 |
